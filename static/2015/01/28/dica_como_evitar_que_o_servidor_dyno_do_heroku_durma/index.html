[Dica] Como evitar que o servidor / Dyno do Heroku durma?

28 January 2015 on Dicas, Projetos, Rails

Se você usa o plano gratuito do Heroku provavelmente sabe das suas limitações. A principal limitação do plano gratuito do Heroku é que muitas vezes o servidor / Dyno pode "dormir" ou "entrar em hibernação" se o seu site não tiver muitas visitas frequentes.

Resolver esse problema é muito simples - pelo menos atualmente - e não exige praticamente nenhum esforço seu. Tenho utilizado o serviço gratuito UpTime Robot e acredito que ele tem feito o seu trabalho muito bem.

O que ele faz é pingar os sites que você configura a cada 5 minutos (no plano gratuito, no pago pode ser em intervalos ainda menores). Caso você não saiba o que é pingar, é como se esse serviço acessa-se o seu site a cada 5 minutos. Por conseguinte, o seu Dyno no Heroku nunca vai entrar em estado de hibernação, pois tecnicamente ele está recebendo visitas. 

O permite até 50 "monitores" que prometem pingar um site cada a cada 5 minutos no plano gratuito, o que é muitoUptime Robot
